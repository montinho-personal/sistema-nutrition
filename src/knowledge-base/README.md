# Knowledge Base

Bibliotecas de conhecimento estruturado do sistema (Documento 03G — Nutrition Knowledge Base).

A inteligência do Montinho Nutrition Strategy **não depende apenas de IA**: nasce da combinação
de base estruturada → regras → algoritmos → IA (Documento 08).

## Bibliotecas previstas

| #   | Biblioteca     | Conteúdo                                                            |
| --- | -------------- | ------------------------------------------------------------------- |
| 1   | `strategies/`  | Estratégias nutricionais (indicações, contraindicações, evidência)  |
| 2   | `foods/`       | Atributos estratégicos de alimentos (complementa o banco TBCA/TACO) |
| 3   | `supplements/` | Suplementos com nível de evidência                                  |
| 4   | `behavior/`    | Comportamento alimentar (gatilhos, estratégias de mudança)          |
| 5   | `adherence/`   | Estratégias de aderência (marmitas, viagens, eventos...)            |
| 9   | `protocols/`   | Protocolos (platôs, baixa aderência, excesso de fome...)            |
| 10  | `evidence/`    | Afirmações científicas com nível de evidência                       |

## Regras

- Separar sempre: alimentos, estratégias, suplementos, regras, protocolos — **nunca misturar**
  (Documento 11).
- Todo conteúdo é versionado no Git e evolui **sem reescrever o software** (Documento 03G).
- Dados nutricionais seguem a prioridade: **TBCA → TACO → bases internacionais → estimativa
  apenas quando inevitável** (Documento 00).
