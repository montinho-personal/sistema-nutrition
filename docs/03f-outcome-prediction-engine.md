# DOCUMENTO 03F — Montinho Nutrition Strategy

## Outcome Prediction Engine (OPE)

---

| Campo             | Valor                    |
| ----------------- | ------------------------ |
| **Classificação** | Core Intelligence Module |
| **Prioridade**    | Máxima                   |

> Este documento define **como o software estima resultados futuros**.
>
> O objetivo nunca será prometer resultados.
> O objetivo será **construir expectativas realistas e orientar a estratégia**.

---

## Missão

Responder continuamente:

> "Qual resultado é mais provável para este aluno se seguir a estratégia proposta?"

---

## Filosofia

O sistema **nunca fará promessas**. O sistema trabalhará com **cenários**, baseados em:

- Aderência.
- Objetivo.
- Rotina.
- Histórico.
- Estratégia.
- Treino.
- Sono.
- Composição corporal.
- Evolução.

## Princípio

O software deverá mostrar **probabilidades**. Nunca certezas.

---

## Cenários

Sempre construir:

| Cenário         | Condição                                           |
| --------------- | -------------------------------------------------- |
| **Conservador** | Caso a aderência seja parcial.                     |
| **Esperado**    | Caso a estratégia seja seguida conforme planejado. |
| **Otimista**    | Caso exista excelente aderência.                   |

---

## Variáveis Analisadas

- Objetivo
- Peso
- Prazo
- Histórico
- Fome
- Sono
- Treino
- Passos
- Rotina
- Composição corporal
- Estratégia nutricional
- Velocidade escolhida
- Disponibilidade
- Ambiente alimentar
- Estresse

---

## Análise de Viabilidade

O sistema deverá responder:

- A meta é fisiologicamente plausível?
- O prazo é compatível?
- O ritmo é sustentável?
- O risco é aceitável?

### Exemplo

```
Aluno: perder 15 kg — Prazo: 60 dias
  ↓
O sistema deverá responder:
  Meta extremamente agressiva.
  Probabilidade baixa.
  Maior risco de abandono.
  Maior risco de perda muscular.
  Maior risco de efeito sanfona.
  ↓
Depois sugerir:
  Prazo alternativo.
  Estratégia alternativa.
```

---

## Probabilidade

Sempre apresentar a probabilidade de sucesso em escala:

- Muito alta
- Alta
- Moderada
- Baixa
- Muito baixa

**Nunca utilizar linguagem absoluta.**

---

## Principais Fatores Positivos

O software deverá listar. Exemplos:

- Boa rotina.
- Treino consistente.
- Alta motivação.
- Boa ingestão proteica.
- Bom suporte familiar.
- Boa disponibilidade.

## Principais Fatores Negativos

- Pouco sono.
- Compulsão.
- Viagens.
- Álcool.
- Baixa organização.
- Pouco tempo.
- Estresse.
- Baixa ingestão de proteína.

## Fatores Modificáveis

O sistema deverá destacar **quais fatores podem ser melhorados rapidamente**. Exemplo:

- Mais passos.
- Melhor sono.
- Planejamento semanal.
- Maior proteína.
- Preparação das refeições.

---

## Linha do Tempo

O software deverá gerar projeções para:

- 30 dias
- 60 dias
- 90 dias
- 120 dias
- 180 dias

**Sempre como projeção. Nunca como garantia.**

---

## Platôs

O sistema deverá prever:

- Quando poderá ocorrer desaceleração.
- Quais fatores normalmente causam isso.
- Como minimizar.

---

## Expectativas

Antes do início, o software deverá explicar:

- Primeiras semanas.
- Perda inicial.
- Possíveis oscilações.
- Retenção hídrica.
- Adaptação.
- Mudanças de fome.
- Mudanças de desempenho.

---

## Indicadores

A cada acompanhamento, atualizar automaticamente:

- Probabilidade.
- Novo cenário.
- Nova previsão.
- Nova estratégia.

---

## Alertas

Sempre avisar quando a estratégia estiver ficando **incompatível com a realidade**. Exemplo:

- Meta muito rápida.
- Baixa aderência.
- Mudança de rotina.
- Eventos futuros.

## Comunicação

Nunca utilizar linguagem alarmista. Sempre utilizar linguagem **educativa**.

---

## Ajustes

Quando a projeção mudar, explicar:

- O que mudou.
- Por que mudou.
- Qual será a nova estratégia.

---

## Transparência

Sempre deixar claro: as projeções dependem de —

- Aderência.
- Treino.
- Rotina.
- Sono.
- Eventos inesperados.
- Resposta individual.

---

## Resultado

Ao finalizar, o software deverá entregar:

- Viabilidade da meta.
- Probabilidade de sucesso.
- Cenário conservador.
- Cenário esperado.
- Cenário otimista.
- Principais riscos.
- Principais oportunidades.
- Plano para aumentar as chances de sucesso.

---

## Princípio Final

O Outcome Prediction Engine existe para **alinhar expectativas**.

Ele nunca promete resultados. Ele **aumenta a qualidade das decisões**.

Quanto melhor o diagnóstico e maior a aderência, mais confiáveis serão as projeções.

> Essa deverá ser sua única missão.
