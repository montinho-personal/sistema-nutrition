# DOCUMENTO 03G — Montinho Nutrition Strategy

## Nutrition Knowledge Base (NKB)

---

| Campo             | Valor                 |
| ----------------- | --------------------- |
| **Classificação** | Core Knowledge Module |
| **Prioridade**    | Máxima                |

> Este documento define toda a **Base de Conhecimento** do Montinho Nutrition Strategy.
>
> O sistema nunca deverá depender apenas do modelo de IA.
> Grande parte da inteligência deverá vir de uma **base estruturada de conhecimento**.

---

## Missão

Criar uma biblioteca inteligente capaz de apoiar todas as decisões do software.

A IA deverá **consultar essa biblioteca antes de sugerir estratégias**.

---

## Filosofia

O conhecimento deverá ser:

- Organizado.
- Modular.
- Atualizável.
- Auditável.
- Baseado em evidências.
- Sempre que possível, **independente do modelo de IA utilizado**.

## Estrutura

A Base de Conhecimento será composta por **bibliotecas independentes**.

Cada biblioteca poderá evoluir sem alterar as demais.

---

## Biblioteca 1 — Estratégias Nutricionais

Cada estratégia deverá possuir:

- Nome
- Descrição
- Objetivo
- Quando utilizar
- Quando evitar
- Perfil ideal
- Perfil inadequado
- Benefícios
- Limitações
- Riscos
- Nível de evidência
- Probabilidade média de aderência
- Complexidade
- Tempo esperado para adaptação

**Exemplos:** déficit moderado, déficit agressivo, manutenção, superávit, recomposição, jejum,
alta frequência alimentar, baixa frequência alimentar, contagem de macros, método das porções,
alta proteína, baixo carboidrato, cetogênica, refeeds, Diet Break, Mini Cut, Lean Bulk.

- Congela bem
- Portabilidade

- Melhores combinações
- Objetivos em que costuma ser útil

## Consulta à Base

Antes de responder, o sistema deverá consultar automaticamente:

- Estratégias
- Alimentos
- Suplementos
- Casos semelhantes
- Protocolos
- Evidências

---

## Evolução

A Base de Conhecimento deverá permitir adicionar novos alimentos, novas estratégias, novos
suplementos, novos estudos e novos protocolos — **sem necessidade de reescrever o software**.

---

## Auditoria

Toda decisão deverá ser **rastreável**. O sistema deverá conseguir responder:

- De onde veio esta recomendação?
- Qual biblioteca foi utilizada?
- Qual o nível de evidência?

---

## Governança

Sempre que duas informações entrarem em conflito, priorizar nesta ordem:

1. Segurança.
2. Evidência científica.
3. Aderência.
4. Realidade do aluno.

---

## Princípio Final

A inteligência do Montinho Nutrition Strategy não deverá depender apenas da IA.

Ela deverá nascer da combinação entre **conhecimento estruturado, evidências científicas,
experiência prática e raciocínio estratégico**.

> A Base de Conhecimento será um patrimônio do software e deverá crescer continuamente ao longo dos anos.
