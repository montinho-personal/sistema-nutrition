# DOCUMENTO 03D — Montinho Nutrition Strategy

## Strategic Priority Matrix (SPM)

---

| Campo             | Valor                    |
| ----------------- | ------------------------ |
| **Classificação** | Core Intelligence Module |
| **Prioridade**    | Máxima                   |

> Este documento define **como o software identifica quais problemas devem ser resolvidos primeiro**.
>
> O objetivo do SPM é impedir que o sistema tente corrigir tudo ao mesmo tempo.
> Toda estratégia deverá começar atacando o **maior gargalo**.

---

## Filosofia

Nem todos os problemas possuem o mesmo impacto. O software deverá identificar quais mudanças
produzirão o maior resultado com o menor esforço possível. O foco será:

- Maior impacto.
- Menor complexidade.
- Maior aderência.
- Maior sustentabilidade.

---

## Missão

Responder sempre:

> "Qual é a próxima melhor decisão para este aluno?"

Não:

> "O que ainda pode ser melhorado?"

---

## Princípio 80/20

O sistema deverá identificar automaticamente quais **20% das mudanças** têm potencial para gerar
aproximadamente **80% dos resultados**.

---

## Identificação dos Gargalos

Após o diagnóstico, o sistema deverá listar todos os gargalos encontrados. Exemplo:

- Baixa ingestão proteica.
- Fome intensa à noite.
- Pouco sono.
- Consumo elevado de álcool.
- Poucas frutas.
- Muitas refeições por delivery.
- Baixa ingestão de fibras.
- Longos períodos em jejum involuntário.
- Final de semana desorganizado.
- Pouco planejamento.

---

## Classificação dos Gargalos

Cada gargalo deverá receber notas em:

- Impacto fisiológico.
- Impacto psicológico.
- Impacto na aderência.
- Impacto na performance.
- Impacto na composição corporal.
- Facilidade de resolução.
- Tempo necessário.
- Custo.
- Urgência.

---

## Matriz de Prioridade

Cada problema deverá ser posicionado em uma matriz:

|                   | **Baixa Dificuldade** | **Alta Dificuldade**    |
| ----------------- | --------------------- | ----------------------- |
| **Alto Impacto**  | 🟢 Prioridade Máxima  | 🟡 Planejamento gradual |
| **Baixo Impacto** | 🔵 Melhoria futura    | ⚪ Baixa prioridade     |

**Regra:** nunca tentar corrigir tudo. **Priorizar.**

---

## Exemplo

```
Aluno:
  Dorme 5 horas.
  Come proteína insuficiente.
  Bebe refrigerante diariamente.
  Treina pesado.
  ↓
O sistema calcula:
  1º Maior impacto imediato → dormir mais
  2º Segundo maior impacto  → aumentar proteína
  3º Terceiro               → reduzir refrigerante

Não inverter.
```

---

## Gargalo Primário

O sistema deverá obrigatoriamente definir o maior gargalo, com:

```
Maior gargalo
  ↓ Justificativa
  ↓ Plano de ação
  ↓ Impacto esperado
```

### Gargalos Secundários

Também deverão ser registrados. Mas tratados apenas quando fizer sentido.

---

## Motor de Pequenas Vitórias

O sistema deverá buscar mudanças que aumentem rapidamente a **confiança do aluno**. Exemplos:

- **Trocar refrigerante comum por zero** → alta chance de sucesso, grande economia calórica, baixa dificuldade.
- **Adicionar uma fruta** → pouco esforço, grande benefício.
- **Preparar marmitas** → grande impacto, maior aderência.

---

## Efeito Dominó

O sistema deverá procurar mudanças que naturalmente facilitem outras mudanças. Exemplo:

```
Dormir melhor
  ↓ Mais energia
  ↓ Treino melhora
  ↓ Menor fome
  ↓ Melhor humor
  ↓ Maior aderência
  ↓ Melhores resultados
```

---

## Custo-Benefício

Cada intervenção deverá receber uma nota em:

- Resultado esperado.
- Tempo.
- Custo.
- Dificuldade.
- Aderência.

---

## Plano de Implementação

O sistema nunca deverá entregar vinte mudanças simultaneamente. Criar etapas:

- Fase 1
- Fase 2
- Fase 3
- Fase 4

---

## Regra da Simplicidade

Sempre perguntar:

> Existe uma solução mais simples?

Se sim → utilizar.

---

## Plano B

Para cada prioridade deverá existir uma alternativa. Exemplo — aluno não consegue cozinhar:

```
Plano A → Marmitas
Plano B → Restaurante por quilo
Plano C → Shake estratégico
Plano D → Delivery inteligente
```

---

## Reavaliação

A cada acompanhamento, a matriz deverá ser **recalculada**:

- Novos gargalos poderão surgir.
- Problemas antigos poderão desaparecer.
- As prioridades deverão mudar automaticamente.

---

## Resultado

Ao final da análise, o sistema deverá entregar:

- Maior gargalo.
- Maior oportunidade.
- Maior risco.
- Mudança de maior impacto.
- Mudança mais fácil.
- Mudança mais urgente.
- Mudança que pode esperar.

---

## Princípio Final

O Montinho Nutrition Strategy nunca tentará criar a **dieta perfeita**.

Ele criará a **próxima melhor decisão possível** para aquele aluno.

> Essa filosofia deverá orientar todas as estratégias do software.
